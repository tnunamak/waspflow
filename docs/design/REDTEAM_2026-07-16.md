# Red-team engagement — model-selection control loop (HEAD 9248a04, 2026-07-16)

Bet-the-company hardening pass. 5 written invariants (I1–I5), one adversarial lane
per invariant in isolated read-only worktrees, synthesis judge, then **every surviving
finding independently reproduced by the maintainer** before it counts. First run was
voided (codex content-filter blocked the adversarial framing — a tooling lesson, not a
result); re-run on Claude general-purpose agents.

## Verdicts

| Invariant | Verdict | Confirmed findings |
|-----------|---------|--------------------|
| I1 attestation-truth | VIOLATED | grok eligible-with-unobserved-effort (P1) |
| I2 selection-honesty | HOLDS | (only a dropped CR-whitespace nit) |
| I3 escalation-crash-safety | PARTIAL | prepared-phase orphan not gated (P2) |
| I4 receipt-durability | VIOLATED | 3 findings (P1, P1, P1) |

Eight of nine attack classes against I1 held (forged log content, alias false-match,
accept-runtime laundering, CAS-across-escalation, provider fallback all defeated) — the
attestation design is sound; the grok effort gap is a fail-open asymmetry vs codex's
fail-closed on the same condition. I2 held cleanly (model passed literally everywhere;
explicit choice never blocked, never substituted). The confirmed defects, reproduced:

### F1 (I4, P1) — lane receipt duplicated across the append→marker crash window
`lib/artifacts.sh:668→674`. The final `lane` receipt appends (668) then sets
`receipt_emitted` separately (674); a crash between, or a failed `receipt.json` copy
(670), leaves the row durable but the marker unset → next reap's guard (583) re-appends.
Reproduced: 2 `receipt_kind:"lane"` rows for one `lane_uuid`. The **segment** path already
defends this exact window (JSONL dedup); the lane path doesn't — an inconsistency in one
file, not a design choice.

### F2 (I4, P1) — segment dedup defeated by any malformed line
`lib/artifacts.sh:529`. The existence check parses the whole raw ledger with `jq -e`;
one torn/malformed line anywhere makes it exit non-zero → the dedup misses → duplicate
`lane_segment` row for the same `(lane_uuid, index)`. Reproduced. Compounds with F1/F3
(a crash can produce the malformed line that then breaks dedup).

### F3 (I4, P1) — torn last line silently drops the next receipt
`lib/artifacts.sh:511`. Append is `printf '%s\n'` with no leading-newline guard; if the
ledger's last line lacks a trailing newline (F1's crash can cause this), the new receipt
glues onto it → both rows unparseable → silent loss. Reproduced.

### F4 (I1, P1) — grok eligible with an unobserved effort
`lib/providers/grok.sh:337`, `lib/artifacts.sh:618`. The effort-mismatch branch is guarded
`-n "$effort"`; when `summary.json` has no `reasoning_effort`, the branch is skipped and
`match` stays true → `stats_eligible:true` receipt attributing `effort:high` to a lane
whose effort was never observed. Codex fails CLOSED on the identical condition
(`no-settings-event → unknown → ineligible`); SCHEMAS_V1 says grok attests BOTH axes.
Reproduced end-to-end. Frequency depends on effort-less summaries (design comment claims
they don't occur), but the eligibility logic is unconditionally wrong when they do.

### F5 (I3, P2) — prepared-phase transition orphaned, not gated
`lib/escalation.sh:367` sets `escalate_failed` leaving `phase=prepared`;
`escalate_transition_requires_resolution` (48-54) excludes `prepared`; so reap/revise
don't refuse. Narrow: prepared is pre-commit (no arm switch, no launch, no leak), so no
double-launch — but a still-journaled resumable transition can be reaped or crossed by
revise. Fix: clear `pending_transition` when abandoning at prepared (make state and the
recovery predicate consistent).

### F6 (I5, P2) — gitignored oracle deps escape the checkpoint fingerprint
`lib/artifacts.sh:266` uses `ls-files --others --exclude-standard`, excluding gitignored
files. A verify oracle that depends on gitignored state (built artifacts, installed deps,
`.env`, gitignored fixtures) can change without changing the fingerprint → reap reuses a
stale green checkpoint and stamps `verified`. Narrower than it reads (needs
gitignored-dependent oracle), but it is the verify-honesty invariant's core claim.

## Dropped (honest non-findings)
- I2 `grep -qxF` CR/whitespace false-block: nit, low confidence, author-acknowledged high
  false-positive risk — catalog slugs come from `jq -r` and won't carry CR. Not a
  demonstrated violation.

## Confidence register (post-engagement, pre-fix)
- **~80%** the control loop's *core* honesty invariants (I1 attestation truth, I2
  no-silent-substitution, I5 no-false-verified) hold for the common path. The confirmed
  defects are real but each needs a specific triggering condition (crash mid-append,
  malformed ledger line, effort-less grok summary, gitignored oracle dep).
- Residual gaps not closed by static/hand-seeded attack: live TUI-resume seams under real
  provider sessions; true concurrent-reap races (only reasoned, not run under load);
  suite load-sensitivity as an independent hardening item.
- **Not claimed**: full command-surface excellence (the excellence-audit phase is separate
  and not yet run).

## Disposition
F1–F4 are the bettable batch (they share the receipt subsystem and each has a
deterministic oracle). F5–F6 are P2 correctness gaps with clear fixes. All fixed in the
follow-up commit with regression tests; re-reviewed maker≠judge before merge.
